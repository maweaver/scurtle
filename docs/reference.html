These are the commands available from within Turtle Script

MOVE/DRAW <distance>

Moves the turtle a specified distance in the direction he's facing. If using DRAW, leaves a line; otherwise, simply moves with no visible changes.

LEFT/RIGHT/POINT <degrees>

Changes the turtle's heading. LEFT and RIGHT both move the turtle relative to the direction he currently faces, and POINT points him in a certain direction, regardless of his previous direction.

COLOR <n|+/-n|color>

Changes the color for the next line. The color can be set in one of three ways:

  • COLOR <n> sets the color to one of nine colors:
    1. Black
    2. White
    3. Red
    4. Orange
    5. Yellow
    6. Green
    7. Cyan
    8. Blue
    9. Purple
  • COLOR <+/-n> adds n to the current color index to choose a new color relative to the existing one.
  • COLOR <name> sets the color to one of the items in the list above using its name according to the list.

TRANSPARENT <n|+/-n>

Changes the amount of transparency in the drawing. Transparency can range from 0 to 100, with 0 being completely opaque and 100 being completely transparent. Transparency can be set to an absolute value by supplying a number, or relative to its current value by prepending a plus or minus sign.

REMEMBER/GOBACK

REMEMBER stores the turtle's current position and direction, and GOBACK returns to the last position. More than one location can be REMEMBER'd; in this case, GOBACK returns to the most recently-remembered position.

REPEAT <iterations>/NEXT

The instructions between the REPEAT and NEXT commands are executed iterations number of times. REPEAT commands can be nested.

# <label>/GO <label>/RETURN

Provides a way to create functions in turtle script. # labels a location in the script. GO moves the current instruction to the given label, and RETURN resumes execution after the GO statement.

LET <variable> <value>

Assigns a value to a variable. A variable is a single word which represents a value, and a value is a number to assign to it.

IF <expr1> <comparator> <expr2>/ELSE/ENDIF

Compares two expressions. The comparator can be one of <, <=, =, >, >=, !=.. If the condition is true, then the lines from the IF to the next ELSE/ENDIF statements are executed. If not, the code from the next ELSE, if present, to the following ENDIF is executed.

PUSH <expr>/POP <variable>

Push stores a value onto the stack. Pop removes the latest value pushed and stores it in a variable.

LSRULE <ruleName> <generates>

Creates an L-system rule with the given name. If a label is defined, the commands inside that routine define the action of the L-system rule. Generates is a list of rule names to replace this rule in the next iteration. If there is no action for this rule, no corresponding label is needed. If there is only an action, and the rule generates itself, no LSRULE command is needed.

LSRUN <iterations> <startingRule>

Runs an L-system for the given number of iterations, starting with the given start rule.
